# /issue $ARGUMENTS

## 사용법
/issue          열린 이슈 전체 목록 확인 후 처리
/issue [번호]   특정 이슈 번호만 처리

## 실행 순서

### 1. 이슈 조회
MCP로 open 상태 이슈 목록 가져오기.
$ARGUMENTS 있으면 해당 번호만, 없으면 전체 목록 출력 후 처리 순서 확인 요청.

### 2. 의존성 확인
이슈 본문에 `참고:` 항목이 있으면 선행 이슈 완료 여부 먼저 확인.
선행 이슈가 미완료면 해당 이슈 건너뛰고 다음으로 이동.

### 3. 타입 판별 (제목의 [] 기준)
| 제목 타입 | 처리 경로 | 승인 게이트 |
|---|---|---|
| [bug] | /fix 패턴 | 생략 가능 |
| [feat] | planner 에이전트 → /add 패턴 | 필수 |
| [remove] | /fix 패턴 + 데이터 영향 확인 | 필수 |
| [refactor] | /fix 패턴 | 생략 가능 |

타입 표기 없으면 처리 전 타입 확인 질문.

### 4. 이슈별 처리 절차
1. 이슈 본문 읽기
2. 섹션명 → grep 키워드로 변환해서 위치 파악
3. [feat] / [remove] 는 [PLAN] 제시 후 승인 대기
4. 승인 후 수정 진행
5. /verify 실행
6. 커밋: `closes #번호` 포함해서 /git 실행
7. 다음 이슈로 이동

### 5. 완료 보고
- 처리 완료 이슈 번호 + 커밋 해시
- 미처리 이슈 목록 (의존성 미충족 또는 잔여)
