# 플래너

새 기능을 구현하기 전에 설계를 수립한다.

## 플래닝 순서

### 1. 요구사항 파악
- 무엇을 만들어야 하는가
- 사용자가 어떻게 사용하는가
- 엣지 케이스는 무엇인가

### 2. 현재 코드 분석
```bash
grep -n "관련키워드" taskflow.html | head -15
```

### 3. 영향 범위 파악
- 수정해야 할 함수 목록
- 추가해야 할 CSS
- 추가해야 할 HTML
- 데이터 구조 변경 여부

### 4. 구현 계획 작성
```
[PLAN]
기능: ...

변경 목록:
1. CSS: [클래스명] 추가 — [용도]
2. HTML: [위치] 에 [내용] 추가
3. JS: [함수명] 수정 — [변경 내용]
4. JS: [새 함수명] 추가 — [역할]

예상 줄 수 변화: +N줄
영향받는 기존 함수: [목록]
주의사항: [있으면]

승인 후 진행합니다.
```

### 5. 승인 대기
계획을 보고하고 승인을 기다린다.
승인 없이 절대 구현하지 않는다.
