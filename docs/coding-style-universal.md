# 코딩 스타일 규칙

> 출처: `docs/Coding_sytle.md` (C/C++ 기반) — 프로젝트 전 언어에 적용

**핵심 원칙**: 명확성과 가독성. 코드는 최대한 간단하게, 주석은 최대한 자세하게.

---

## 1. 파일 헤더

모든 소스 파일 맨 위에 헤더 블록 작성.

```
/******************************************************************************
FILE NAME   : 파일명.확장자
DESCRIPTION : 파일 역할 한 줄 요약
DATA        : YYYY-MM-DD
******************************************************************************/
```

**언어별 적용:**
- JS/CSS: `/* ... */` 블록 사용
- Go: `// FILE NAME: ...` 라인 블록 사용
- SQL: `-- FILE NAME: ...` 라인 블록 사용

## 2. 네이밍

| 형식                   | 예시              | 적용 대상                                      |
| ---------------------- | ----------------- | ---------------------------------------------- |
| `snake_case`           | `task_count`      | 지역 변수, 함수 인자 (Go, JS)                  |
| `g_SnakeCase`          | `g_UserList`      | 전역 변수 (JS: `let tasks`, Go: package-level) |
| `PascalCase`           | `GetTaskList`     | 함수·메서드, 타입, 클래스                      |
| `SCREAMING_SNAKE_CASE` | `MAX_RETRY_COUNT` | 상수, 매크로, 환경변수                         |
| `camelCase`            | `taskCount`       | JS 객체 속성, JSON 키                          |

**공통 규칙:**
- 이름 길이 최대 30자
- 이름만 보고 역할 추측 가능해야 함
- 단일 문자 변수 금지 (`i` → `ii`, `j` → `jj`)
- 임시 변수는 `ii`, `jj`, `tmp`, `len` 허용
- `a`, `b`, `c` 절대 금지

## 4. 함수 주석

함수마다 헤더 주석 작성. 간단한 유틸은 한 줄 주석 허용.

```
/******************************************************************************
FUNCTION    : 함수명
DESCRIPTION : 역할 설명
PARAMETERS  : type 인자명 - 설명
              type 인자명 - 설명
RETURNED    : 반환값 설명 (void면 생략)
******************************************************************************/
```

**언어별 변형:**
```js
// JavaScript
/******************************************************************************
FUNCTION    : RenderTaskList
DESCRIPTION : 업무 목록을 DOM에 렌더링
PARAMETERS  : tasks[] - 렌더링할 업무 배열
              filter  - 적용할 필터 객체
RETURNED    : void
******************************************************************************/
function RenderTaskList(tasks, filter) { ... }
```

```go
// Go
/******************************************************************************
FUNCTION    : GetTaskByID
DESCRIPTION : ID로 단일 업무 조회
PARAMETERS  : ctx   context.Context - 요청 컨텍스트
              id    string          - 업무 ID
RETURNED    : *Task, error
******************************************************************************/
func GetTaskByID(ctx context.Context, id string) (*Task, error) { ... }
```

## 8. 언어별 추가 규칙

### JavaScript (src/js/)
- `const` → 재할당 없는 변수, `let` → 재할당 있는 변수, `var` 금지
- 객체·배열 선언 후 한 줄에 하나의 속성
- 템플릿 리터럴은 80자 초과 시 변수로 분리
- CSS 클래스명: `kebab-case`

### Go (backend/)
- 패키지 레벨 변수 `var` 블록으로 그룹화
- 에러 처리: `if err != nil` 반드시 체크, 무시 금지
- 수신자명: 타입 첫 글자 소문자 (`func (t *Task) ...`)

### CSS (src/css/)
- 선택자 하나당 속성 한 줄
- 관련 속성 그룹: 박스모델 → 위치 → 타이포 → 색상 순

### SQL (migrations/)
- 키워드 대문자: `SELECT`, `FROM`, `WHERE`
- 컬럼·테이블명: `snake_case`
- 각 절은 새 줄에서 시작
